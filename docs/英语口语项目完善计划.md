LinguaCoach 英语口语项目完善计划
================================

目标与范围
----------
目标是提升“愿意聊下去”的核心体验，同时解决语音识别首句丢失与对话节奏问题。该计划聚焦现有架构下的最小化改动，不引入新抽象，不改变现有 WebSocket 消息格式，不修改 archive/ 中任何文件。

现状问题摘要
------------
1) 语音识别
- ASR 建连存在空窗期，首句识别延迟与吞字风险。
- 长连接可能带来回声，需要平衡缓冲与噪音策略。

2) 对话体验
- 追问密度过高、封闭式问题偏多，用户容易进入“被动应答”。
- 解释词义后继续追问，认知负担叠加。
- 对话前期响应慢，影响开场情绪。

3) 用户心理负担
- 频繁评分易造成压力，应改为“隔段时间评价+夸与建议并存”。

4) 难度与个性化
- 不同水平用户对话风格相似，缺乏难度与节奏差异化。

成功标准
--------
- 用户主动结束率下降，平均轮次上升。
- 前 5 分钟平均响应延迟下降并稳定。
- 评估与鼓励内容让用户“更愿意继续说”，主观评价提升。
- 话题延展性更好，用户回答长度与信息量提升。

原则与约束
----------
- 只改动完成目标必需的模块与 Prompt。
- 避免引入新架构或独立评估服务。
- 不改变现有 WebSocket 消息格式。
- 不在 openrouter_audio.py 中写 Prompt。

改进方向与模块计划
------------------
1) Prompt 设计（prompts/templates.py）
- 系统提示词：强调“开放式提问、避免复述、降低问责式追问、保持轻松交流”。
- 用户提示词：加入“追问节奏控制规则”，限制连续追问次数，鼓励话题转换与共情反馈。
- 初始问题：减少“事实型问句”，改为更易展开的开放式场景问题。

2) 统一处理器（services/unified_processor.py）
- 解析结果中增加“对话节奏标记”的支持（如：当前轮次、用户回答长度、是否应转话题）。
- 将“短答/重复答/拒答”转为不同的引导策略，避免机械追问。

3) 评分与反馈（services/unified_processor.py 为主，prompts/templates.py 为辅）
- 评分展示改为“分段/间隔式”，不在每轮输出，由调用逻辑控制是否生成/返回评分字段。
- Prompt 仅作为软约束，提示模型“非每轮评分”，但不依赖模型自控节奏。

4) 难度与引导（prompts/templates.py）
- 依据用户水平与回答长度调整问题复杂度与词汇。
- 允许中英文混用，中文比例高时轻提醒英文，但不打断交流流畅性。

5) 响应时延与首句识别（speech.py / funasr_service.py / streaming_voice_endpoint.py）
- 评估长连接与重连策略在首句识别上的表现差异。
- 在可控范围内加缓冲或填充策略，保证首句捕获率。

----------
- 首句识别成功率、对话前 5 分钟平均响应时间。

- 语音识别优化可能引入回声，需要实测确认。
- 评分节奏调整需与前端展示逻辑一致。
