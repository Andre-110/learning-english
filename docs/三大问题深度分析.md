## 1️⃣ 问题1: 上下文长度 vs 延迟关系

### 🔍 现状分析

**当前项目的日志**:
```python
# services/gpt4o_pipeline.py:469-473
history_len = len(conversation_history) if conversation_history else 0
logger.info(f"[LLM] 对话历史: {history_len} 条, 预览: {preview_str}")
```

**问题**: 只记录了 `history_len`，**没有记录 LLM 耗时与 history_len 的关系**。

**UserGenie 的做法** (`openai-voice-relay.ts`):
```typescript
// 性能指标追踪
interface PerformanceMetrics {
  llmStartTime: number;
  llmFirstTokenTime: number;  // TTFT (Time To First Token)
  llmEndTime: number;
}

// 溯源日志：完整记录 ASR → LLM → TTS
private logTrace(sessionId: string, stage: 'LLM_INPUT', data: any) {
  console.log(`${prefix} Messages count: ${data.messagesCount}`);
  console.log(`${prefix} System prompt length: ${data.systemPromptLength} chars`);
}
```

### 📈 理论分析

根据 UserGenie 的延迟测试报告:
- **LLM 平均延迟**: 2017ms (GPT-4o)
- **token 数量影响**: 每增加 1000 tokens，预计增加 100-300ms 延迟

**预期关系**:
| 对话轮数 | 预估 Token 数 | 预估 LLM 延迟 |
|---------|--------------|--------------|
| 1-2 轮 | ~500 tokens | ~1500ms |
| 5-6 轮 | ~1500 tokens | ~2000ms |
| 10+ 轮 | ~3000 tokens | ~3000ms+ |

### 🔧 解决方案

**Step 1: 添加延迟追踪日志**

在 `services/gpt4o_pipeline.py` 的 `chat()` 方法中添加:

```python
# 记录上下文 token 估算
estimated_tokens = sum(len(m.get('content', '')) // 4 for m in messages)
logger.info(f"[LLM] 📊 上下文分析: history_len={history_len}, estimated_tokens={estimated_tokens}")

# 在响应完成后记录
elapsed_ms = int((time.time() - start_time) * 1000)
logger.info(f"[LLM] ⏱️ 延迟分析: history_len={history_len}, tokens≈{estimated_tokens}, latency={elapsed_ms}ms")
```

**Step 2: 创建分析脚本**

```bash
# 收集数据后分析
grep "延迟分析" nohup.out | awk -F'[=,]' '{print $2,$4,$6}' > latency_data.csv
# 使用 Python/Excel 绘制 token 数 vs 延迟曲线
```

**Step 3: 根据分析结果调整**

当前设置 (`models/conversation_memory.py`):
```python
max_short_term_tokens: int = 1200  # 已从 2000 降低
summary_trigger_turns: int = 3     # 已从 5 降低
```

如果分析发现延迟与 token 数强相关，可进一步降低。

## 3️⃣ 问题3: ASR 热备份连接断开

### 🔍 现状分析

**当前项目的 ASR 重连机制** (`services/deepgram_asr.py`):
```python
self._auto_reconnect = True
self._max_reconnect_attempts = 3
self._reconnect_delay = 1.0  # 递增延迟

# 重连时保存转录
self._preserved_transcript_before_reconnect = self._full_transcript.strip()
```

**前端处理** (`frontend/src/stores/conversation.js`):
```javascript
case 'asr_reconnecting':
  console.log('🔄 [ASR] 重连中...')
  recordAnomaly('ASR_RECONNECTING', `ASR 重连中 (尝试 ${data.attempt})`)
  break
case 'asr_reconnected':
  console.log('✅ [ASR] 连接已恢复')
  break
```

**问题**: 
1. **没有追踪断连原因** - 不知道是网络问题、服务端超时还是其他
2. **没有持久化日志** - 无法事后分析

### 📊 UserGenie 的做法

**1. 详细的断连日志**:
```typescript
// 记录异常事件
anomalyEvents: Array<{ type: string; message: string; timestamp: number; data?: any }>;

// 断连时记录详情
console.log(`🔄 [Doubao ASR] Unexpected close, code=${code}, reason=${reason}`);
console.log(`📝 [Doubao ASR] Preserving transcript before reconnect: "${preservedTranscript}"`);
```

**2. 5分钟会话缓存** (支持断网重连):
```typescript
private static readonly SESSION_CACHE_TIMEOUT_MS = 5 * 60 * 1000; // 5分钟内可重连恢复对话

// 断开时保留上下文
if (existingState && existingState.isDisconnected) {
  if (disconnectedDuration < SESSION_CACHE_TIMEOUT_MS) {
    console.log(`🔄 [Session Resume] Restoring session, history: ${existingState.conversationHistory.length} messages`);
  }
}
```

**3. 溯源日志**:
```typescript
private logTrace(sessionId: string, stage: 'ASR' | 'LLM_INPUT' | 'LLM_OUTPUT' | 'TTS', data: any) {
  // 完整记录每个阶段，方便 grep 查找
  console.log(`📋 [TRACE][${sessionId}][${stage}]`);
}
```

### 🔧 解决方案

**Step 1: 添加 ASR 断连详细日志**

修改 `services/deepgram_asr.py`:
```python
async def _on_close(self, close_event):
    # 🆕 记录断连详情
    logger.warning(f"[Deepgram] ❌ 连接断开: code={close_event.code}, reason='{close_event.reason}'")
    logger.warning(f"[Deepgram] 状态: is_processing={self._is_processing}, transcript_len={len(self._full_transcript)}")
    
    # 分析断连原因
    if close_event.code == 1000:
        logger.info("[Deepgram] 正常关闭")
    elif close_event.code == 1006:
        logger.warning("[Deepgram] ⚠️ 异常关闭 (网络问题或服务端超时)")
    elif close_event.code == 1008:
        logger.error("[Deepgram] ❌ 策略违规 (可能是 API Key 问题)")
```

**Step 2: 添加异常事件追踪**

在 `api/gpt4o_pipeline_endpoint.py` 添加:
```python
# 类似 UserGenie 的 anomalyEvents
anomaly_events = []

def record_asr_anomaly(event_type: str, details: dict):
    anomaly_events.append({
        "type": event_type,
        "timestamp": datetime.utcnow().isoformat(),
        "details": details
    })
    logger.warning(f"[ASR Anomaly] {event_type}: {details}")
```

**Step 3: 创建 ASR 健康监控脚本**

```bash
#!/bin/bash
# monitor_asr.sh - 实时监控 ASR 异常
tail -f nohup.out | grep -E "(ASR.*断开|reconnect|异常关闭|ANOMALY)"
```
