# qwen-omni Prompt 遵循能力分析报告

## 📊 测试概述

- **测试用例**: 20个中英文混杂的句子
- **输入方式**: 音频输入（通过 TTS 生成）
- **成功处理**: 20/20 (100%)

---

## 1. 中英文转录准确性 ⚠️ **严重问题**

### 📈 统计结果

- **中文保留率**: 5/20 (25.0%) ❌
- **中英文混杂保留率**: 6/20 (30.0%) ❌

### ❌ 主要问题

#### 问题1: 中文被误听成拼音（15个错误）

| 原文 | 转录结果 | 错误类型 |
|------|----------|----------|
| `足球` | `tsuuchio` | 中文被误听成拼音 |
| `土豆` | `potato` | 中文被翻译成英文（正确但不符合要求） |
| `番茄` | `tomato` | 中文被翻译成英文（正确但不符合要求） |
| `篮球` | `lanchio` | 中文被误听成拼音 |
| `请` | `Ching` | 中文被误听成拼音 |
| `的` | 缺失 | 中文部分丢失 |
| `困难` | `quinnan` | 中文被误听成拼音 |

**根本原因**：
- OpenAI TTS 不支持中文，中文部分发音不标准
- qwen-omni 模型将不标准的中文发音误听成拼音
- **不符合 prompt 要求**：prompt 明确要求"保留原始语言"

#### 问题2: 中文部分被翻译（不符合要求）

- `土豆` → `potato`（虽然意思正确，但不符合"保留原始语言"的要求）
- `番茄` → `tomato`（同上）

#### 问题3: 中文部分完全丢失

- `请` → 缺失
- `的` → 缺失

### ✅ 正确案例（5个）

1. `请继续详细地介绍一下Messi的故事` → 完整保留中文
2. `这个football player真的很amazing` → 部分保留中文
3. `我想了解more about football` → 部分保留中文
4. `请continue介绍Messi的childhood` → 部分保留中文
5. `请explain more about Messi的playing style` → 部分保留中文

## 2. 建议质量分析 ⚠️ **中等**

### 📈 统计结果

- **总修正数**: 38条
- **冗余修正**: 6条（15.8%）
- **中文处理率**: 5/20 (25.0%) ❌

### ❌ 问题

#### 问题1: 冗余修正（6个）

修正建议中出现了 `original == corrected` 的情况，例如：
- `they are delicious` → `they are delicious`（说明：This part is correct!）
- `Messi` → `Messi`（说明：No correction needed）

**不符合 prompt 要求**：prompt 明确要求"如果无需修正，corrections 必须为空数组 []"

#### 问题2: 中文处理不足（75%未处理）

只有 25% 的用例正确处理了中文部分，75% 的用例虽然包含中文，但修正建议中没有针对中文的修正。

**示例**：
- 用例1: `I want to talk some thing about 足球player` 
  - 转录: `tsuuchio player`
  - 修正建议中没有将 `tsuuchio` 修正回 `足球`

#### 问题3: 分数合理性

大部分用例分数合理（25-40分），符合中英文混杂的预期。

## 5. 关键问题总结

### 🔴 严重问题

1. **中文转录错误率高（75%）**
   - 中文被误听成拼音（`足球` → `tsuuchio`）
   - 中文被翻译成英文（`土豆` → `potato`）
   - **不符合 prompt 要求**：应该保留原始语言

2. **中文处理不足（75%未处理）**
   - 虽然识别到中文，但修正建议中没有针对中文的修正

### 🟡 中等问题

3. **冗余修正（15.8%）**
   - 将"无需修正"的内容放入修正列表
   - **不符合 prompt 要求**：应该使用空数组

---

## 6. 改进建议

### 高优先级

1. **修复中文转录问题**
   - ✅ **使用支持中文的 TTS**（edge-tts 的中文语音）
   - ✅ **优化 prompt**：强调"准确转录中文部分，不要翻译或拼音化"
   - ✅ **后处理优化**：识别常见拼音错误并修正

2. **修复冗余修正问题**
   - ✅ **优化 prompt**：明确要求"如果无需修正，corrections 必须为空数组 []"
   - ✅ **后处理过滤**：在 ResponseParser 中过滤 `original == corrected` 的修正

3. **提高中文处理率**
   - ✅ **优化 prompt**：明确要求"如果转录中有拼音化的中文，必须修正回中文"
   - ✅ **示例增强**：在 prompt 中添加中文处理的示例

### 中优先级

4. **提高转录准确性**
   - 使用更好的 TTS 服务（支持中英文混合）
   - 优化音频质量

---

## 7. 结论

**总体评价：⭐⭐⭐ (3/5) - 需要改进**

qwen-omni 模型在**评分合理性**方面表现良好，但在**中英文转录准确性**和**建议质量**方面存在明显问题。

**主要优势**：
- ✅ 评分基本合理
- ✅ 能识别语法错误
- ✅ 修正建议详细

**主要劣势**：
- ❌ 中文转录错误率高（75%）
- ❌ 中文处理不足（75%未处理）
- ❌ 冗余修正问题（15.8%）

**最关键的问题**：中文部分被误听成拼音，**严重不符合 prompt 要求**。这主要是由于 TTS 不支持中文导致的，需要改用支持中文的 TTS 服务。
