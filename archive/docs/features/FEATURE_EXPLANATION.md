# 功能说明文档

## 问题修复说明

### 1. 系统回答用户问题 ✅

**问题描述：**
作为英语交互系统，系统之前只生成问题，不回答用户的问题（如"你是谁"）。

**解决方案：**
- 在 `core/conversation.py` 的 `process_user_response` 方法中添加了问题检测逻辑
- 使用 `_is_user_asking_question()` 方法检测用户是否在问问题
- 使用 `_generate_answer_to_question()` 方法生成回答
- 如果用户问了问题，系统会先回答，然后继续生成新的学习问题

**触发时机：**
- 每次用户发送消息后，系统会检测是否是问题
- 如果是问题，会先回答，然后再生成新的学习问题

**示例：**
- 用户："你是谁"
- 系统："I'm LinguaCoach, your English learning assistant. I help you practice English through conversations and provide feedback on your speaking skills."
- 系统："Can you tell me about your favorite food? (你能告诉我你最喜欢的食物吗？)"

### 2. 用户画像和最新评估更新时机 ✅

**问题描述：**
用户画像和最新评估何时触发更新不明确。

**更新时机：**
- **最新评估**：每次用户回答后立即更新
  - 触发位置：`core/conversation.py` 的 `process_user_response()` 方法
  - 更新内容：综合分数、CEFR等级、强项、弱项
  - 前端显示：在收到 `assessment` WebSocket消息时自动更新

- **用户画像**：每次用户回答后同步更新
  - 触发位置：`core/conversation.py` 的 `process_user_response()` 方法（第163-197行）
  - 更新内容：综合分数、CEFR等级、对话轮数、用户兴趣
  - 前端显示：在收到 `assessment` WebSocket消息时自动更新（包含 `user_profile` 数据）

**数据流程：**
1. 用户发送音频 → STT转录
2. 系统处理用户回答 → 执行评估 → 更新用户画像
3. 通过WebSocket发送 `assessment` 消息（包含评估结果和用户画像）
4. 前端接收消息 → 调用 `updateAssessment()` → 更新UI显示

**前端代码位置：**
- `static/app.js` 第250-254行：处理 `assessment` 消息
- `static/app.js` 第550-607行：`updateAssessment()` 函数更新UI

### 3. 性能测试功能说明 ✅

**功能描述：**
性能测试功能用于测量整个语音对话流程的各个阶段的响应时间，帮助识别性能瓶颈。

**使用方法：**
1. 点击"开始对话"按钮创建对话
2. 点击"🚀 开始性能测试"按钮启动测试
3. 点击"开始录音"按钮进行录音
4. 完成一次完整的对话流程后，查看性能数据

**测量的指标：**
- **录音时间**：用户录音的持续时间
- **音频发送**：从停止录音到音频数据发送完成的时间
- **首次转录**：从发送音频到收到第一个实时转录结果的时间
- **最终转录**：从发送音频到收到最终转录结果的时间
- **评估处理**：从最终转录到收到评估结果的时间
- **问题生成**：从评估到收到新问题的时间
- **TTS生成**：从问题生成到TTS音频播放完成的时间
- **总处理时间**：从音频发送到TTS完成的全部时间

**性能评级：**
- 🟢 **快速**（绿色）：响应时间在预期范围内
- 🟡 **中等**（黄色）：响应时间稍慢，但可接受
- 🔴 **缓慢**（红色）：响应时间较慢，可能需要优化

**代码位置：**
- `static/app.js` 第678-856行：性能测试相关代码
- `static/app.js` 第727-856行：`updatePerformanceMetrics()` 函数计算和显示性能指标

**注意事项：**
- 性能测试需要在对话开始后才能使用
- 每次测试会重置之前的性能数据
- 测试过程中按钮会显示"⏱️ 测试进行中..."状态

## 技术实现细节

### 问题检测逻辑

系统使用以下方法检测用户是否在问问题：

1. **关键词匹配**：检测常见问题模式（who, what, when, where, why, how等）
2. **疑问词检测**：检查是否以疑问词开头
3. **标点符号**：检测问号（?、？）和中文疑问词（吗、呢）

### 回答生成

当检测到用户问题时：
1. 使用LLM服务生成自然回答
2. 回答简洁（1-2句话）
3. 支持中英文问题
4. 回答后自然过渡到下一个学习问题

### 评估更新流程

```
用户回答 → STT转录 → 评估服务 → 更新用户画像 → WebSocket发送 → 前端更新UI
```

### 性能测试流程

```
开始测试 → 录音开始 → 录音结束 → 音频发送 → 转录 → 评估 → 问题生成 → TTS → 显示结果
```
