# 前端性能测试功能使用说明

## 功能说明

前端已添加性能测试功能，用户可以实时查看各个处理环节的耗时。

## 使用方法

1. **开始对话**
   - 输入用户ID
   - 点击"开始对话"按钮

2. **启动性能测试**
   - 点击"🚀 开始性能测试"按钮
   - 性能测试区域会显示出来

3. **进行录音**
   - 点击"开始录音"按钮
   - 说一段话（建议5-10秒）
   - 点击"停止录音"按钮

4. **查看性能数据**
   - 性能指标会实时更新
   - 显示各个处理环节的耗时
   - 最后显示总体性能评估

## 性能指标说明

### 显示的指标

1. **录音时间**：用户录音的时长
2. **音频发送**：从停止录音到音频发送完成的时间
3. **首次转录**：收到第一个实时转录结果的时间
4. **最终转录**：收到最终转录结果的时间
5. **评估处理**：LLM评估用户回答的时间
6. **问题生成**：生成下一个问题的时间
7. **TTS生成**：文本转语音的时间
8. **总耗时**：从开始录音到TTS播放完成的总时间

### 性能评级

- **绿色（fast）**：性能优秀
- **黄色（medium）**：性能良好
- **红色（slow）**：性能较慢

### 性能评估标准

- **总耗时 < 5秒**：✅ 性能优秀
- **总耗时 5-10秒**：⚠️ 性能良好
- **总耗时 > 10秒**：❌ 性能较慢

## 性能优化建议

根据测试结果，可以：

1. **如果转录时间慢**：
   - 检查网络连接（如果使用API）
   - 考虑使用本地STT服务（FunASR）

2. **如果评估处理慢**：
   - 检查LLM API配置
   - 考虑使用更快的模型

3. **如果TTS慢**：
   - 检查TTS服务配置
   - 考虑使用更快的TTS服务

## 注意事项

- 性能测试会在每次录音时自动记录数据
- 测试完成后，可以再次点击"开始性能测试"进行新的测试
- 性能数据会实时更新，无需刷新页面
